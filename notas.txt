GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO agrigestor_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO agrigestor_user;

-- Importante: permisos para tablas que se creen en el futuro
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
    GRANT ALL ON TABLES TO agrigestor_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
    GRANT ALL ON SEQUENCES TO agrigestor_user;

-- Salir
\q

Agregar proyecto a GitHub
# Inicializa el repositorio
git init

# Verifica qué archivos se van a subir
# (confirma que .env NO aparece en la lista)
git status

# Agrega todos los archivos
git add .

# Verifica una vez más lo que se va a commitear
git diff --staged --name-only

# Primer commit
git commit -m "feat: inicio del proyecto AgriGestor

- Configuración de Django 5 + FastAPI + PostgreSQL
- Modelos: Usuario, Parcela, Cultivo, AplicacionInsumo
- Autenticación JWT
- CRUD completo de Parcelas, Cultivos e Insumos
- Módulo de Reportes con KPIs agrícolas"

Desde este momento cada avance que hagas en el proyecto debe seguir este flujo:
# 1. Ver qué cambió
git status

# 2. Agregar los cambios
git add .

# 3. Hacer commit con mensaje descriptivo
git commit -m "feat: descripción de lo que agregaste"

# 4. Subir a GitHub
git push

Convención de mensajes que se ve profesional en el historial:
feat: nueva funcionalidad
fix: corrección de error
refactor: mejora de código sin cambiar funcionalidad
docs: cambios en documentación
test: agregar o modificar tests
